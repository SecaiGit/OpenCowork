# 上下文压缩 / Context Compression

当对话历史增长时，OpenCowork 会自动压缩上下文以防止超出模型的 token 限制。

## 两种压缩策略 / Strategies

### 预压缩（Pre-compress）

在每次 Agent 循环迭代前执行，清理较早的大块工具内容：

- 保留最近的工具结果，避免影响当前轮推理
- 将较早且内容较长的工具结果、thinking 块和图片附件替换为占位符，减少 token 消耗
- 不改变对话结构，保留工具调用记录

### 完整压缩（Full Compress）

当 token 数超过有效触发线时触发：

1. 调用 LLM 对历史对话生成摘要
2. 用摘要替换旧消息
3. 保留最近 N 条消息不压缩

## 触发条件 / Triggers

压缩会先确定本轮使用的上下文长度和阈值：

- **模型上下文长度优先**：模型配置了 `contextLength` 时，优先使用模型自己的窗口。
- **未知模型使用全局默认**：模型没有上下文长度时，使用设置页里的默认上下文长度。
- **模型阈值优先于全局默认**：模型配置了 `contextCompressionThreshold` 时，优先使用模型阈值；否则使用全局默认压缩阈值。

```typescript
const effectiveWindow = contextLength - reservedOutputBudget

const trigger = Math.min(
  Math.floor(effectiveWindow * threshold),
  effectiveWindow - autoBufferTokens
)

const preCompressTrigger = Math.min(
  Math.floor(effectiveWindow * preCompressThreshold),
  effectiveWindow - preCompressBufferTokens,
  trigger - preCompressGapTokens
)
```

## 时间戳过滤 / Timestamp Filtering

插件消息解析器使用 15 分钟时间戳过滤，丢弃过期消息，防止重复处理历史消息。
